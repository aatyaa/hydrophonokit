## 🏗️ File Management System Design

### Core Principles

1. **Separation of Concerns:**
   - Core modules in `hydrophonokit/` package
   - Tests in `tests/` directory
   - Documentation in project root
   - Build artifacts excluded from git

2. **Module Organization:**
   - `analyzer.py` → Material analysis
   - `generator.py` → Workspace generation
   - `verifier.py` → Integrity validation
   - `postprocessor.py` → Results analysis
   - `physics.py` → Physical constants
   - `templates.py` → VASP templates
   - `inspector.py` → VASP output validation

3. **Advanced Analysis Modules:**
   - `elastic.py` → Elastic constants
   - `eos.py` → Equation of State
   - `qha.py` → Quasi-Harmonic Approximation
   - `anharmonic.py` → Phonon lifetimes/linewidths
   - `h2_molecule.py` → H2 gas properties
   - `h_storage.py` → Hydrogen storage

4. **CLI Architecture:**
   - `cli.py` → Command definitions
   - `__main__.py` → Entry point for `python -m hydrophonokit`
   - `__init__.py` → Package exports + version

## 🔧 File System Operations

### Input/Output Patterns

#### 1. **Source Directory (VASP Input)**
```
source_dir/                          # User-provided relaxed VASP directory
├── CONTCAR                         # Required
├── INCAR                           # Required
├── OUTCAR                          # Required
├── POTCAR                          # Required
└── vasprun.xml                     # Optional (bandgap extraction)
```

**Validation:**
- `cli.py::_require_source()` checks for CONTCAR, INCAR, OUTCAR, POTCAR
- Raises `SourceDirectoryError` if missing

#### 2. **Workspace Directory (Generated Output)**
```
workspace/                           # Generated by `hydrophonokit build`
├── 00_unitcell/
│   └── POSCAR                      # Reference unit cell
├── 01_born/                        # (If insulator)
│   ├── POSCAR
│   ├── INCAR                       # DFPT Born calculation
│   ├── POTCAR
│   └── run.sh                      # SLURM script
├── 02_displacements/
│   ├── perfect/
│   │   └── POSCAR                  # Perfect supercell
│   ├── disp-001/
│   │   ├── POSCAR                  # Displaced structure
│   │   ├── INCAR                   # Force calculation
│   │   ├── KPOINTS
│   │   ├── POTCAR
│   │   └── run.sh                  # SLURM script
│   ├── disp-002/
│   └── ...
├── phonopy_disp.yaml               # Phonopy displacement data
├── submit_born.sh                  # Master script for Born
├── submit_all_disps.sh             # Master script for displacements
└── PHONON_PLAN.txt                 # Summary report
```

**Validation:**
- `verifier.py::SystemVerifier` checks all required files
- Raises `VerificationError` if critical files missing

#### 3. **Results Directory (Post-processing Output)**
```
workspace/phonon_results_TIMESTAMP/  # Generated by `hydrophonokit postprocess`
├── phonon_band_structure.png       # Band structure plot
├── phonon_dos_partial.png          # Partial DOS plot
├── phonon_thermodynamics.png       # F(T), S(T), Cv(T)
├── thermodynamic_properties.dat    # Raw data
├── band.yaml                       # Band structure data
├── Phonon_Analysis_Report.html     # HTML summary
├── FORCE_CONSTANTS                 # Force constant matrix
└── BORN                            # Born charges (if applicable)
```

**Special Cases:**
- If H present: `H_mode_analysis.png`
- Elastic: `elastic_constants.json`
- QHA: `qha_results.png`, `qha_data.json`
- Anharmonic: `anharmonic_results.png`, `anharmonic_data.json`
- H-storage: `h_thermodynamics.png`, `vant_hoff.png`, `h_storage_data.json`

## 🔒 Error Handling & File Validation

### Exception Hierarchy

```python
# cli.py
HydroPhonoKitError(Exception)           # Base exception
├── SourceDirectoryError            # Missing/invalid source
└── WorkspaceDirectoryError         # Missing/invalid workspace

# generator.py
GenerationError(Exception)          # Workspace generation failed

# verifier.py
VerificationError(Exception)        # Integrity check failed
└── (reports warnings separately)
```

### File Validation Flow

```
1. User provides --source DIR
   ↓
2. _require_source() validates CONTCAR, INCAR, OUTCAR, POTCAR
   ↓
3. analyzer.py reads files
   ↓
4. generator.py creates workspace
   ↓
5. verifier.py validates workspace integrity
   ↓
6. User uploads to HPC
   ↓
7. postprocessor.py reads completed workspace
   ↓
8. Results saved to workspace/phonon_results_TIMESTAMP/
```

## 📋 File Naming Conventions

### Python Modules
- **Snake case:** `analyzer.py`, `generator.py`, `postprocessor.py`
- **Descriptive:** `h2_molecule.py`, `h_storage.py`
- **No abbreviations:** `elastic.py` (not `elast.py`)

### Output Files
- **Underscores:** `phonon_band_structure.png`
- **Descriptive:** `thermodynamic_properties.dat`
- **Timestamps:** `phonon_results_20260412_143052/`

### Test Files
- **Prefix:** `test_` + module name
- **Match module:** `test_elastic.py` → tests `elastic.py`

## 🗂️ Directory Management

### Created During Operation

| Command | Creates | Location |
|---------|---------|----------|
| `build` | Workspace directory | User-specified `--output` |
| `postprocess` | Results subdirectory | Inside workspace/ |
| `qha` | Plot + JSON | User-specified `--output` |
| `anharmonic` | Plot + JSON | User-specified `--output` |
| `hstorage` | Plots + JSON | User-specified `--output` |

### Automatic Cleanup

- `generator.py` implements **rollback** on failure:
  - Tracks all created directories in `_created_dirs`
  - On exception, removes all tracked directories
  - Prevents partial workspaces

## 🔍 Potential Issues & Solutions

### Issue 1: Missing Source Files

**Problem:** User provides directory without CONTCAR/INCAR/OUTCAR/POTCAR

**Solution:**
```python
def _require_source(args):
    missing = [f for f in ['CONTCAR', 'INCAR', 'OUTCAR', 'POTCAR']
               if not os.path.exists(os.path.join(src, f))]
    if missing:
        raise SourceDirectoryError(f"Missing: {', '.join(missing)}")
```

### Issue 2: Partial Workspace Generation

**Problem:** Generator fails midway, leaving incomplete workspace

**Solution:**
```python
def generate(self):
    try:
        # ... generation code ...
    except Exception as e:
        self._rollback()  # Removes all created dirs
        raise GenerationError(...) from e
```

### Issue 3: Workspace Verification

**Problem:** User uploads incomplete workspace to HPC

**Solution:**
```python
verifier.verify_integrity()  # Checks ALL required files
# Raises VerificationError if anything missing
```

### Issue 4: Results Overwrite

**Problem:** Postprocessing overwrites previous results

**Solution:**
```python
# Timestamped output directory
stamp = datetime.datetime.now().strftime('%Y%m%d_%H%M%S')
self.output_dir = os.path.join(workspace, f'phonon_results_{stamp}')
```

### Issue 5: Cross-Platform Path Issues

**Problem:** Windows vs Linux path separators

**Solution:**
- Use `os.path.join()` everywhere
- Use `pathlib.Path` for new code
- Test on Windows and Linux

### Issue 6: Large File Handling

**Problem:** OUTCAR can be 100+ MB, vasprun.xml can be 1GB+

**Solution:**
- Parse OUTCAR line-by-line (not load all into memory)
- Use `re` for efficient pattern matching
- Consider streaming XML parsing for vasprun.xml

## 🛡️ Best Practices

### For Users

1. **Always validate source directory:**
   ```bash
   hydrophonokit analyze -s ./relax_out  # Check before build
   ```

2. **Verify workspace before upload:**
   ```python
   from hydrophonokit.verifier import SystemVerifier
   v = SystemVerifier(workspace, n_disp)
   v.verify_integrity()
   ```

3. **Use unique output directories:**
   ```bash
   hydrophonokit build -s ./relax -o ./phonon_ws_v1
   ```

### For Developers

1. **Use `os.path.join()` for all paths**
2. **Validate file existence before reading**
3. **Use try/except for file operations**
4. **Implement rollback for multi-step operations**
5. **Use timestamped output directories to avoid overwrites**
6. **Log file operations for debugging**
